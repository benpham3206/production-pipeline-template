# Operations Checklist

## Deployment
- [ ] Deployment runbook documented
- [ ] Rollback procedure documented and tested
- [ ] Zero-downtime deployment verified
- [ ] Post-deploy smoke test automated

## Incident Response
- [ ] Incident severity levels defined (P1-P4)
- [ ] On-call rotation configured
- [ ] Escalation path documented
- [ ] Communication templates (status page, internal, customer)
- [ ] Post-incident review process

## Data Recovery
- [ ] Data backup schedule documented
- [ ] Recovery procedure tested (at least quarterly)
- [ ] RPO and RTO defined and achievable
- [ ] Data export capability for compliance

## Maintenance
- [ ] Dependency updates reviewed monthly
- [ ] Security patches applied within SLA (critical: 24h, high: 72h)
- [ ] Certificate renewal automated or tracked
- [ ] Log rotation configured
- [ ] Database maintenance (vacuum, reindex) scheduled

## Documentation
- [ ] Runbook for each alert
- [ ] Architecture diagram current
- [ ] On-call handoff procedure
- [ ] Disaster recovery plan tested annually
